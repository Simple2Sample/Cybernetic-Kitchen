Oppgave 2

I denne oppgaven skal vi ta for oss Activity-Selection Problem fra kapittel 16.1. Nedenfor har vi gitt tre (grådige) algoritmer som er forsøk på å løse dette problemet. Input er S og F, med henholdsvis starttidene og sluttidene for aktivitetene. si og fi er starttid og slutttid for element i.

I Wangchang(S,F) antar vi at aktivitetene er sortert etter synkende sluttid.

Wangchang(S,F):
n ← length(s)
A ← {1}
j ← 1
for i ← 2 to n:
do if fi ≥ sj then
 A ← A ∪ {i}
 j ← i
 return A

I Mimblewimble(S,F) antar vi at aktivitetene er sortert etter synkende starttid.

Mimblewimble(S,F):
n ← length(s)
A ← {1}
j ← 1
for i ← 2 to n:
do if fi ≤ sj then
 A ← A ∪ {i}
 j ← i
 return A

I Nitwit(S,F) antar vi at aktivitetene er sortert etter økende varighet.

Nitwit(S,F):
n ← length(s)
A ← {1}
for i ← 2 to n:
ok ← true
 for a in A:
 do if i collides with a:
 ok ← false
 do if ok:
 A ← A ∪ {i}
 j ← i
 return A

"if i collides with a" sjekker her om aktivitet i og a overlapper.

a) Hva er det grådige valget som gjøres i Nitwit? (6.8 %)

Den konstruerer et huffman-tre, og velger neste aktivitet ut fra binærkoden til starttiden.
 Den velger alltid den aktiviteten som varer kortest blant de som ikke overlapper med en tidligere valgt aktivitet
 Den velger den aktiviteten som passer best inn blant de som allerede er valgt
 Den velger alltid den aktiviteten som varer kortest

b) Hvilket grådig valg gjøres i Mimblewimble? (6.8 %)

Mimblewimble velger den aktiviteten som til starter sist, men som ikke slutter etter at den forrige valgte starter.
 Mimblewimble velger den aktiviteten som slutter sist, fordi brukere av algoritmen trenger god tid på de aktivitetene de vil foreta seg.
 Mimblewimble velger den aktiviteten som starter først, men ikke før den forrige aktiviteten slutter
 Mimblewimble velger den aktiviteten som slutter først, og lar det dermed være mest mulig plass igjen til andre aktiviteter.

Din kommentar:
LF-forklaring: Her må vi huske på at huffmantrær er fulle: enhver node har enten null eller to barn. I det "verste" tilfellet vil alle de interne nodene ha en løvnode som det ene barnet og en intern node som det andre barnet, med unntak av den nederste interne noden, hvor begge barna er løvnoder. Et huffmantre for n tegn skal ha n løvnoder. Den første er på nivå 1, den andre på nivå 2 osv. - den siste og nest siste noden er dermed på nivå n-1.
